## 1. 제품 개요 & 한 줄 피치

**한 줄 피치**: "이름을 지웠습니다. 당신의 눈을 믿으세요." — 국적·유명세·선입견을 모두 가린 채, 실제 월드컵 선수들의 스탯과 히트맵만 보고 나만의 베스트11을 뽑고, 마지막에 그들의 정체를 마주하는 드라마.

**제품 정의**: 실제 월드컵(2018·2022) 선수 데이터를 익명 카드로 제시하고, 사용자가 감독이 되어 데이터만으로 11명을 선발·배치한 뒤, "정체 공개" 연출로 자신이 뽑은 선수들의 실명·정체와 실제 대회 성적을 대조하는 싱글플레이 웹 경험.

**왜 이 컨셉인가**:
- 대회 예시 4종(전술보드/AI추천/what-if시뮬/교체지시) 어디에도 없음 → **참신성 30점 정면 공략**.
- 실시간 경기 엔진이 없어 3주 안에 버그 없는 안정적 작동 달성 가능 → **완성도 25점 확보 유리**.
- 데이터가 화면(카드·히트맵)에 직접 드러나고 그 데이터로 판단·배치하는 인터랙션이 명확 → **일관성 20점 강함**.
- "편견을 지운 판단 → 정체 반전"이라는 감정 훅 → **감동 25점 공략**.

## 2. 타겟 사용자 & 핵심 가치

**1차 타겟 = 심사위원 (첫 방문·짧은 체류·계정 없음)**. 이것이 모든 설계의 제약이자 나침반이다.

| 사용자 | 맥락 | 핵심 가치 |
|---|---|---|
| 심사위원 (최우선) | 무설치·무가입·처음 방문, 여러 출품작 연속 심사로 인내심 짧음 | 30초 내 "무엇을 하는 것인지" 체감 + 3~5분 안에 완주 가능한 완결된 감정 곡선 |
| 축구 팬 (2차) | 자기 안목을 시험하고 싶음, 공유 욕구 | "내 눈이 맞았나?"의 확인 쾌감 + 자랑할 결과 카드 |

**핵심 가치 명제**:
1. **안목 시험** — 편견 없이 데이터로만 판단했을 때 나는 진짜 좋은 선수를 알아보는가?
2. **반전의 감동** — 무명인 줄 알았는데 레전드, 스타인 줄 알았는데 의외의 인물이었다는 순간.
3. **완주 보장** — 심사위원이 이탈 없이 감정 곡선 끝(여운)까지 도달한다.

**온보딩 요건 (심사 직격)**:
- 진입 후 **30초 내에** 게임의 목적·조작 방식이 체감되어야 한다 (별도 튜토리얼 페이지 강제 없이, 첫 화면 카피+시각으로).
- 첫 클릭까지의 마찰 최소화: 로딩·회원가입·설정 화면 없이 바로 코어 루프 진입.

## 3. 코어 루프 상세 (화면 흐름)

싱글 세션 완결형. 단계별 흐름:

**S0. 진입 / 훅 (Hook)**
- 첫 화면: 익명 카드 몇 장 + 선언 카피("이름을 지웠습니다. 당신의 눈을 믿으세요.").
- CTA 하나("스카우팅 시작") → 즉시 S1. 대회(2018 or 2022)·포메이션 선택이 필요하면 여기서 최소 1스텝으로.

**S1. 선수 카드 탐색 / 비교 (Flow)**
- 익명 카드 풀 제시: 각 카드는 **실명·국적·사진 없이** 포지션군, 스파이더차트(핵심 스탯), 히트맵, 소수의 핵심 수치만 노출. 실루엣/도형으로 표현.
- 카드 상세 열람(뒤집기/확대), 2장 이상 비교 뷰.
- 포지션별로 후보를 좁혀가는 탐색.

**S2. 포메이션 배치 (드래그)**
- 필드 위 포메이션 슬롯(11칸)에 카드를 **드래그로 배치**.
- 포지션 제약(GK 1명 필수, 라인별 정원 등) 및 — 채택 시 — 예산/밸런스 제약을 시각적으로 안내.
- 배치 중 실시간으로 라인업 상태(빈 슬롯, 포지션 미스매치) 피드백.

**S3. 확정 (Commitment)**
- 11칸이 규칙에 맞게 채워지면 "정체 공개" CTA 활성화.
- 확정 직전 "돌이킬 수 없습니다" 뉘앙스의 긴장 유발 마이크로카피(가벼운 브레이크).

**S4. 정체 공개 연출 (Climax) — 제품의 심장**
- 11명의 정체가 **하나씩 순차적으로** 드라마틱하게 공개(카드 플립 + 이름·국적·정체 등장).
- 각 공개마다 반전 강도(무명↔레전드)를 감성적으로 강조. 페이싱·사운드(옵션)·모션으로 클라이맥스 조성.

**S5. 결과 / 안목 점수 (Resonance)**
- 내가 뽑은 XI에 대한 "안목 점수"/등급 + 내러티브 코멘트.
- 내 XI가 **실제 그 월드컵에서 어떻게 됐는지** 대조(후보4 요소 차용: 소속팀 성적/우승·탈락 등 공개 사실).
- 하이라이트: 가장 의외의 픽, 최고의 픽 등 스토리 요소.

**S6. 공유 / 재도전 (Replay)**
- 결과를 이미지 카드로 공유(내 XI + 안목 점수).
- "다시 하기"(다른 대회/다른 카드 풀) 즉시 재진입 → 리플레이 훅.

## 4. 기능 요구사항 (유저스토리 + 수용기준)

우선순위: **MUST**(3주 내 반드시, 이것만으로 완결 데모 가능) / **SHOULD**(시간 되면, 감동·완성도 강화) / **COULD**(여유 시 폴리시).

### MUST (코어 최소 — 이것만으로 심사 완주 가능해야 함)

**M1. 익명 선수 카드 열람**
- 스토리: 감독으로서, 이름·국적을 모르는 채 선수의 스탯·히트맵만 보고 싶다. 편견 없이 순수 실력으로 판단하기 위해.
- 수용기준:
  - 카드에 실명/국적/사진/로고가 **일절 노출되지 않는다** (실루엣·도형·익명ID만).
  - 각 카드는 포지션군 + 스파이더차트 + 히트맵 + 핵심 수치를 표시한다.
  - 카드를 클릭/탭하면 상세(확대)를 볼 수 있다.

**M2. 포지션 제약 하 드래그 배치**
- 스토리: 감독으로서, 카드를 필드 슬롯에 드래그해 나만의 11명을 짜고 싶다. 감독의 조작감을 느끼기 위해.
- 수용기준:
  - 드래그앤드롭으로 카드를 11칸 포메이션 슬롯에 배치/교체/제거할 수 있다.
  - GK 최소 1명 등 포지션 규칙을 위반하면 시각적으로 막거나 경고한다.
  - 11칸이 규칙대로 채워지기 전엔 확정 CTA가 비활성.
  - 최소 1개 포메이션(예: 4-3-3)은 반드시 지원.

**M3. 정체 공개 연출**
- 스토리: 감독으로서, 확정 후 내가 뽑은 선수들의 진짜 정체를 극적으로 마주하고 싶다. 내 판단의 반전을 체험하기 위해.
- 수용기준:
  - 확정 시 11명의 정체(실명·국적·정체)가 **순차적으로** 애니메이션과 함께 공개된다.
  - 공개 후 각 슬롯에서 익명↔실명 매핑이 명확히 보인다.
  - 연출은 최소한의 모션(플립/페이드/스태거)으로도 "클라이맥스"로 느껴져야 한다.

**M4. 결과 / 안목 피드백**
- 스토리: 감독으로서, 내 선택이 얼마나 좋았는지 점수/코멘트로 확인하고 싶다.
- 수용기준:
  - 공개 후 XI에 대한 정량 점수 또는 등급이 산출·표시된다.
  - 내가 뽑은 선수들의 실제 대회 관련 사실(성적/맥락) 최소 1가지가 대조로 제시된다.
  - 점수 산출 로직은 데이터 기반으로 설명 가능해야 한다(임의값 금지).

**M5. 온보딩 훅**
- 스토리: 첫 방문 심사위원으로서, 진입 즉시 무엇을 하는 게임인지 알고 싶다.
- 수용기준:
  - 첫 화면에 컨셉을 각인하는 카피 + 단일 CTA가 있다.
  - 튜토리얼 없이도 30초 내 첫 배치 조작에 도달 가능.

**M6. 무설치·무가입·무결제 심사 가능 + 출처표기**
- 스토리: 심사위원으로서, 링크만 열면 키·로그인 없이 바로 플레이하고 싶다.
- 수용기준:
  - 회원가입/로그인/결제/외부 키 없이 배포 URL에서 전 기능 동작.
  - StatsBomb 출처표기가 화면 내(예: 푸터/정보)에 상시 노출.
  - 선수 사진·팀 로고 미사용(실명 텍스트 + 실루엣만).

### SHOULD (감동·완성도 강화)

**S1. 카드 비교 뷰** — 2장 이상을 나란히 놓고 스파이더/히트맵 직접 비교. 수용: 최소 2장 동시 비교, 지표 축 정렬 일치.

**S2. 예산/밸런스 제약** — "감독의 고뇌"를 만드는 제약(예산 상한 또는 라인 밸런스). 수용: 제약 위반 시 확정 불가, 남은 예산 실시간 표시. (주의: 예산의 "가치"를 무엇으로 정의할지 = OPEN-QUESTION.)

**S3. 결과 공유 카드** — 내 XI+점수를 이미지로 저장/공유. 수용: 1클릭으로 공유용 이미지 생성, 실명·출처표기 포함.

**S4. 복수 포메이션 / 복수 대회** — 4-3-3 외 포메이션, 2018·2022 선택. 수용: 최소 2포메이션 또는 2대회 중 하나 이상 다양성 제공.

**S5. 반전 강조 연출** — 공개 시 "가장 의외의 픽"을 하이라이트하는 서사. 수용: 무명↔레전드 격차를 지표로 판별해 1장 이상 강조.

### COULD (여유 시 폴리시)

- **C1. 사운드/햅틱 주스** — 플립·배치·공개 효과음.
- **C2. 히트맵 인터랙션** — 히트맵 위 호버로 세부 수치.
- **C3. 리더보드/누적** — (서버 없이) 로컬 기록으로 이전 시도 대조.
- **C4. 다국어(영문)** — 심사는 국내지만 확장 대비.

## 5. 감동 경험 설계 (배점 25점 직접 공략)

**감정 곡선 (진입→몰입→클라이맥스→여운)**

| 단계 | 감정 | 설계 장치 |
|---|---|---|
| Hook (S0) | 호기심·도전 의식 | "이름을 지웠습니다" 선언 카피, 익명 카드의 미스터리, 3초 각인 |
| Flow (S1~S2) | 몰입·고뇌 | 데이터 비교 퍼즐 + 제약(포지션/예산) + 드래그 촉각 피드백 |
| Climax (S4) | 긴장→반전→카타르시스 | 순차 정체 공개, 반전 강조, 페이싱 조절 |
| Resonance (S5~S6) | 만족·자기확인·재도전 욕구 | 안목 점수, 실제 성적 대조, 공유 카드, 즉시 재도전 |

**정체 공개 연출 요구사항 (M3 심화)**:
- **순차 공개**: 한 번에 다 까지 말고 1명씩 스태거링 → 각 반전에 정서적 무게 부여.
- **반전 극대화**: 익명 카드가 준 인상과 실명 사이의 갭이 큰 픽을 시각적으로 강조.
- **되돌릴 수 없음의 긴장**: 확정 직전 짧은 브레이크로 클라이맥스 전 긴장 축적.
- **드라마틱 톤 일관**: 색·타이포·모션이 "감성 연출"로 통일(designer 스펙 대상).

**마이크로 인터랙션 포인트 (조작 직관성 = 감동 배점 항목)**:
- 카드 뒤집기/확대 시 부드러운 전환.
- 드래그 시 스냅·고스트·유효 슬롯 하이라이트.
- 잘못된 배치 거부 시 명확하고 부드러운 피드백(에러가 아니라 안내로 느껴지게).
- 확정 CTA 활성화 순간의 상태 변화 강조.
- 공개 순간의 스태거 애니메이션.

## 6. 심사 배점 매핑

| 기준(배점) | 이를 충족하는 기능/설계 | 어떻게 |
|---|---|---|
| **참신성 30** | 컨셉 자체 + M1(익명 카드) + M3(정체 공개) | 예시 4종에 없는 "익명 데이터 판단→정체 반전" 훅. 심사위원이 처음 보는 접근. |
| **감동 경험 25** | 감정 곡선(§5) + M3 연출 + M2 드래그 + 마이크로 인터랙션 | "내가 감독" 몰입(고뇌하는 선발) + 조작 직관성(드래그·비교) + 반전 카타르시스. |
| **완성도 25** | 실시간 엔진 부재 + 사전집계 데이터(§7) + MUST 최소 코어 | 런타임 리스크 낮음 → 버그 없는 안정 작동. 최소 코어에 집중해 3주 내 완결. |
| **일관성 20** | 데이터가 화면에 직접 드러남 + M1~M4 흐름 | 기획("데이터로만 판단")과 구현(카드·히트맵·점수)이 1:1로 대응. |

## 7. 데이터 요구사항

**출처**: StatsBomb Open Data (GitHub, 무료 JSON), FIFA 남자 월드컵 **2018·2022** 이벤트 단위 데이터.

**핵심 전제**: 우리가 **오프라인에서 사전 집계**해 정적 JSON으로 번들 → 심사자 키 불필요, 런타임 안정. (라이브 API 금지.)

**필요 지표 (선수 카드용 후보 목록 — architect/데이터 작업이 확정)**:
- **공격**: xG, 슛 수, 슛 정확도, 득점, 드리블 성공.
- **패스**: 패스 성공률, 키패스/찬스 크리에이션, 전진 패스, xA(가능 시).
- **수비**: 태클, 인터셉트, 볼 리커버리, 압박 액션.
- **포지션/공간**: 히트맵(위치 밀도), 평균 포지션, 활동 범위.
- **메타(익명)**: 포지션군, 익명ID, 출전 표본 크기(신뢰도 판단용).
- **정체(공개 단계에서만 노출)**: 실명, 국적, 대회, 소속팀 성적/맥락.

**요건**:
- 카드에 쓰는 모든 지표는 실제 StatsBomb 집계값이어야 한다(임의 생성 금지).
- 표본이 적은 선수(소수 경기) 처리 방침 필요 → 최소 출전 임계값 (OPEN-QUESTION).
- 카드 풀 규모(대회당 선수 수)는 탐색 피로도와 균형 → 큐레이션 필요 (OPEN-QUESTION).
- **라이선스 표기**: StatsBomb attribution 상시 노출, 비상업(해커톤) 범위, 선수 사진/팀 로고 미사용.

## 8. 스코프 아웃 (명시적으로 안 하는 것)

- **회원가입/로그인/계정** — 무가입 심사 요건. 안 함.
- **서버 사이드 저장/DB 영속** — 세션 완결형. 결과 저장은(하더라도) 로컬 한정.
- **멀티플레이/대전/실시간 협업** — 싱글 완결.
- **실시간 경기 시뮬레이션 엔진** — 후보3 영역, 완성도 리스크. 안 함.
- **AI 전술 추천/LLM 기능** — 코어 아님. (예시일 뿐 필수 아님.)
- **라이브 데이터 API 연동** — 사전집계 정적 번들로 대체.
- **선수 사진·팀 로고·엠블럼** — IP 리스크. 실명 텍스트+실루엣만.
- **여자 월드컵/기타 대회** — 이번 범위는 남자 2018·2022로 고정.
- **교체·인게임 전술 지시·타임라인 리플레이** — 코어 루프에 없음.
- **결제/유료 기능** — 무결제 요건.

## 9. 성공 기준 & 리스크

**성공 기준**:
- 심사위원이 링크 진입 후 이탈 없이 S0→S6 **완주**한다(3~5분).
- 정체 공개(S4)에서 "반전"의 감정이 실제로 터진다(데모 영상에서 확인 가능).
- 배포 URL에서 무가입·무키로 전 MUST 기능이 **오류 없이** 동작.
- 기획서(7/27)와 최종 구현(8/3)의 인터랙션·데이터 활용이 일치.

**리스크 & 대응**:
| 리스크 | 영향 | 대응 |
|---|---|---|
| StatsBomb 집계 가공량 과다 | 일정 지연 | 대회 1개(예: 2022) MVP 먼저, 나머지는 SHOULD |
| 익명 카드가 "무엇을 비교하는지" 불명확 | 몰입 실패 | 지표 축소·직관적 시각화, designer 스펙 필수 |
| 정체 공개가 밋밋 | 감동 배점 손실 | 연출을 MUST로 못박고 designer 주도, ui-reviewer로 검수 |
| 안목 점수 신뢰성 논란 | 일관성 손실 | 점수 로직을 데이터 기반·설명가능하게 |
| 카드 풀 과다로 탐색 피로 | 이탈 | 대회당 선수 수 큐레이션 |
| 실명 사용의 IP/명예훼손 경계 | 실격 위험 | 사실 스탯만, 비하 표현 금지, 사진/로고 배제 |

## 10. 오픈 퀘스천

### A. architect / 데이터 작업이 정할 것 (오너 개입 불필요)
- 사전집계 파이프라인·번들 포맷·정적 서빙 방식.
- 렌더링 스택(카드·히트맵·스파이더·드래그).
- 히트맵/스파이더 시각화 구현 방식(코드 생성 원칙: 벡터/캔버스).
- 최소 출전 표본 임계값의 기술적 산출.
- 안목 점수 알고리즘의 구체 가중치(데이터 기반 설계).

### B. 오너 결정 필요 (제품/범위)
1. **MVP 대회 범위**: 2018·2022 둘 다 MUST인가, 아니면 하나(예: 2022)를 MUST로 하고 나머지 SHOULD인가? (일정 안전마진 직결)
2. **"예산" 제약 채택 여부**: 감독의 고뇌를 위한 예산/밸런스 제약을 넣을까(SHOULD S2), 넣는다면 예산의 가치 기준을 무엇으로(스탯 종합? 임의 등급?)? — 넣으면 몰입↑·구현량↑.
3. **안목 점수의 "정답" 정의**: 무엇을 "좋은 선발"로 볼 것인가 — (a) 선수 스탯 종합 우수도, (b) 실제 대회 성적/수상, (c) 둘의 혼합? 이게 여운(S5)의 서사를 결정.
4. **여운 단계 성적 대조 범위**: 개별 선수의 소속팀 성적까지 보여줄지, 아니면 "당신의 XI 종합 등급"까지만 할지 (데이터 조달량·서사 깊이 트레이드오프).
