## 0. 디자인 원칙 (5줄)

1. **어둠이 기본, 빛이 강조.** 화면은 야간 경기장. 정보와 CTA만 빛난다. 배경은 눌러 긴장을 만든다.
2. **데이터가 주인공.** 카드의 스파이더/히트맵/수치가 시각 중심. 장식은 데이터를 방해하지 않는다.
3. **익명은 차갑게, 공개는 뜨겁게.** 익명 상태는 무채색+청록(감정 절제), 공개 순간 앰버/포지션색이 터진다(대비로 카타르시스).
4. **모션은 의미다.** 모든 애니메이션은 상태 전이를 설명한다. 장식 모션 금지. transform/opacity만(잰크 회피).
5. **여백이 고급을 만든다.** 촘촘함보다 숨 쉬는 레이아웃. 심사자가 30초 안에 "무엇을"을 읽게.

## 1. 디자인 토큰

### 1.1 색 팔레트 (CSS 변수)

```css
:root {
  /* --- 배경 (야간 경기장, 깊이감 3단) --- */
  --bg-base:      #0A0E14;   /* 최심부 배경 (body) */
  --bg-pitch:     #0D141C;   /* 필드/보드 영역 */
  --bg-vignette:  #06090D;   /* 비네트 가장자리 */

  /* --- 표면 (카드/패널, 위로 갈수록 밝음) --- */
  --surface-1:    #141B24;   /* 기본 패널 */
  --surface-2:    #1C2631;   /* 카드 표면 */
  --surface-3:    #26333F;   /* 호버/부상 카드 */
  --surface-line: #33424F;   /* 구분선/테두리 */

  /* --- 텍스트 --- */
  --text-hi:      #EAF1F7;   /* 제목/핵심 수치 */
  --text-mid:     #A5B4C0;   /* 본문/라벨 */
  --text-lo:      #66757F;   /* 보조/캡션/출처 */
  --text-onAccent:#0A0E14;   /* 앰버 위 텍스트 */

  /* --- 강조: 플러드라이트 앰버 (단일 히어로 강조) --- */
  --accent:       #E8B04B;   /* CTA, 공개 하이라이트, 활성 */
  --accent-hi:    #F6C766;   /* 앰버 밝은 톤 (글로우 코어) */
  --accent-dim:   #8A6A2C;   /* 앰버 비활성/그림자 */
  --accent-glow:  rgba(232,176,75,0.45); /* 발광 rgba */

  /* --- 데이터 청록 (익명/스탯 시각화, 차가운 이성) --- */
  --data:         #4FC3D9;   /* 스파이더 폴리곤/히트맵 하이 */
  --data-dim:     #2A6B78;   /* 히트맵 로우 */

  /* --- 포지션군 4색 (채도 절제, 상호 구분) --- */
  --pos-gk:       #E8B04B;   /* GK  앰버 (수문장=빛) */
  --pos-def:      #5B8DEF;   /* DEF 블루 (냉정) */
  --pos-mid:      #3FB98C;   /* MID 그린 (연결) */
  --pos-fwd:      #E5533C;   /* FWD 레드 (예각) */

  /* --- 상태 --- */
```

포지션 4색은 **채도 65~75%로 눌러** 야간 배경 위에서 서로 구분되되 튀지 않게. 카드 테두리·슬롯 라벨·정체공개 액센트에만 사용(면 전체 채색 금지 = 유치함 방지).

### 1.2 타이포그래피 (Google Fonts, 한글 지원)

```css
/* 로드: Google Fonts */
/* Oswald wght@500..700 (라틴 임팩트/숫자),
   Gothic A1 wght@700;900 (한글 임팩트 — 헤드라인),
   Noto Sans KR wght@400;500;700 (본문/UI),
   Roboto Mono wght@500;700 (데이터 수치) */
/* import 예:
   @import url('https://fonts.googleapis.com/css2?family=Gothic+A1:wght@700;900&family=Noto+Sans+KR:wght@400;500;700&family=Oswald:wght@500..700&family=Roboto+Mono:wght@500;700&display=swap'); */

--font-display: 'Oswald', 'Gothic A1', 'Noto Sans KR', sans-serif; /* 제목·히어로 */
--font-body:    'Noto Sans KR', 'Oswald', sans-serif;              /* 본문·UI */
--font-mono:    'Roboto Mono', 'Noto Sans KR', monospace;          /* 스탯 수치 */
```

> **폰트 결함 이력 (2026-07-06):** 기존 디스플레이 한글 폰트 `Black Han Sans`는 흔한 음절(름/눈/목/등)에서 **notdef 글리프**를 렌더하는 폰트 자체 결함이 확인되어 폐기. `Gothic A1`(완전한 한글 커버리지, 100~900 웨이트 보유)로 대체 — 글리프 리스크 없고 900 웨이트로 임팩트 확보.

**타입 스케일** (1.25 비율, rem 기준 16px):

| 토큰 | size / line-height | weight | 폰트 | 용도 |
|---|---|---|---|---|
| `display-xl` | 64 / 1.0 | 900* | display | 히어로 "이름을 지웠습니다" |
| `display-l`  | 44 / 1.05 | 900* | display | 화면 타이틀, 공개된 실명 |
| `h1` | 32 / 1.15 | 600 | display | 섹션 헤더 |
| `h2` | 24 / 1.2 | 600 | display | 카드 상단 라벨 |
| `h3` | 19 / 1.3 | 600 | body | 소제목 |
| `body` | 16 / 1.55 | 400 | body | 본문 |
| `body-sm` | 14 / 1.5 | 400 | body | 라벨·설명 |
| `caption` | 12 / 1.4 | 500 | body | 출처·메타·태그 |
| `stat-xl` | 40 / 1.0 | 700 | mono | 핵심 수치 강조 |
| `stat` | 20 / 1.1 | 600 | mono | keyStat 값 |
| `overline` | 11 / 1.3 | 600 | body | 대문자 라벨(letter-spacing .12em) |

한글 제목은 Oswald가 라틴만 커버 → 한글 자동 폴백 `Gothic A1`(임팩트 유지). 본문 한글은 Noto Sans KR. 숫자·스탯은 Roboto Mono로 정렬감·데이터 느낌.

**\*웨이트 주의:** `display-xl/l`은 Oswald 라틴이 최대 700이라 브라우저가 900을 합성볼드(가짜)로 렌더할 수 있다. 규칙 — 라틴/숫자 히어로는 Oswald **700**, 한글 히어로 텍스트에는 `font-weight:900`로 `Gothic A1` 900(실제 웨이트 존재)을 태워 임팩트를 낸다. `h1/h2`(display, 600)는 스택 그대로 유지.

## 2. 화면별 레이아웃 스펙 (5화면)

좌표계: 데스크탑 1440 기준, 컨텐츠 1200 컬럼 중앙. 아래는 와이어 구조 + 핵심 치수.

### S0. 온보딩 / 훅 (`features/onboarding`)

```
┌──────────────────────────────────────────────┐  풀블리드, 비네트 배경
│              [상단 미세 로고 BLIND XI]          │  overline, 좌상단 sp-6
│                                                │
│         이름을 지웠습니다.        ← display-xl  │  중앙정렬, max 720
│         당신의 눈을 믿으세요.                   │
│                                                │
│   [익명카드 3장 부채꼴 배치, 살짝 회전/떠있음]   │  카드폭 220, -8°/0°/+8°
│                                                │
│       실명도, 국적도, 사진도 없습니다.          │  body-sm, text-mid
│       데이터만 보고 당신의 베스트11을.          │
│                                                │
│            ▐  스카우팅 시작  ▶  ▌  ← CTA 앰버   │  h 56, px sp-6, glow-soft
│         2022 카타르 · 4-3-3 (기본)  ← 대회칩     │  선택 가능 pill
│                                                │
│      데이터: StatsBomb Open Data  ← caption     │  하단 sp-6, text-lo
└──────────────────────────────────────────────┘
```
- 히어로 카피 진입: display-xl 두 줄 stagger 페이드업(각 0.6s, delay 0.15s).
- 부채꼴 카드 3장은 뒷면(익명 실루엣)만. 마우스 이동에 미세 패럴럭스(±6px).
- CTA는 항상 활성. 대회/포메이션은 pill 토글(기본 2022·4-3-3 선택됨). SHOULD 전엔 2022 고정 표시.
- **상태**: 로딩(데이터 fetch 중)=CTA 자리에 스피너+"선수 풀 불러오는 중"; 에러=재시도 버튼+"데이터를 불러오지 못했습니다".

- 카드 그리드: 카드 220×300, 반응형 열수(1200폭=4열, 900=3열, 600=2열).
- 필터칩: 포지션군 클릭 시 해당 색 활성 테두리, 그리드 필터.
- 우측 패널: 카드 클릭 시 슬라이드인(x:24→0, 0.32s). 비교 트레이는 2장 스파이더 오버레이(두 폴리곤 겹쳐 diff).
- 카드 → 슬롯 드래그를 이 화면에서도 허용(하단 미니보드로 바로 배치 가능) 또는 S2로 이동.
- **상태**: 빈(필터 결과 0)="이 포지션 카드가 없습니다"; 로딩=스켈레톤 카드 6장(shimmer); 에러=상단 배너.

## 4. 정체 공개 연출 시퀀스 (제품의 심장 · 배점 25점 직결)

**컨셉**: 야간 경기장 스포트라이트. 화면 암전, 한 명씩 조명 아래 카드가 뒤집히며 정체가 드러난다. 사운드 없이 **모션·조명·타이밍만으로** 긴장→반전→카타르시스. 스타는 마지막.

### 4.1 전체 타임라인 (11명, 자동 진행 기준)

```
t=0.0s   화면 암전 (0.5s 페이드 to bg-vignette), "정체 공개" overline 등장
t=0.5s   스포트라이트 콘 켜짐 (radial glow 확대 0.4s), 첫 카드 무대 슬라이드인
── 카드 1장 사이클 (약 3.2s) ×11, 아래 4.3 참조 ──
         카드 간 스태거: 이전 카드 퇴장(0.3s) ↔ 다음 진입(0.3s) 크로스
t≈35s    11번째(스타) 공개 후 1.2s 홀드 → "결과 보기" CTA 앰버 등장
```
자동 진행 + 수동 "다음"(클릭 시 현재 홀드 스킵하고 다음). "건너뛰기"=stagger 없이 전원 즉시 플립 후 결과로.

### 4.2 공개 순서 (드라마 곡선)

파이프라인 `fameProxy`/`biggestUpset` 기반으로 store가 순서 배열 제공. 드라마 순서 규칙:

1. **1~4번: 평이한 픽** (fameProxy 중간대) — 워밍업, 리듬 형성.
2. **5~8번: 반전 픽** (익명 인상 ↔ reveal 갭 큰 순으로 점증) — `biggestUpset` 후보를 중반~후반 배치, 에피셋 강조 연출 강화.
3. **9~10번: 준-스타**.
4. **11번(마지막): 최고 스타** (fameProxy 최대) — 최장 홀드(1.2s), 조명 최강, 앰버 글로우 풀.

각 카드의 반전 강도(`|reveal_fame − anonymous_impression|`)에 따라 연출 강도 3단계 스케일(약/중/강 = 글로우·홀드·에피셋 크기 차등).

### 4.3 카드 1장 플립 시퀀스 (Framer Motion, 프레임 단위)

현재 카드 1장 사이클 ≈ 3.2s. transform/opacity만 사용.

```
[진입]   0.00–0.30s  카드 익명면으로 무대 진입: x: 60→0, opacity 0→1, ease-out
                     스포트라이트 콘이 카드로 좁혀짐(scale 1.1→1.0)
[정적]   0.30–0.80s  익명면 홀드(0.5s) — "이게 누구지?" 긴장. 미세 부유(y ±2, loop)
[플립]   0.80–1.40s  rotateY 0→180°, ease-inout 0.6s
                     · 0.80–1.10 (0→90°): 익명면, 밝기 감소(brightness 1→0.4)
                     · 1.10s (90°): 컨텐츠 스왑(익명→공개), backface-hidden
                     · 1.10–1.40 (90→180°): 공개면, 포지션색 오버레이 페이드인
[임팩트] 1.40–1.70s  공개 순간: 앰버 글로우 팝(box-shadow 0→glow→soft, 0.3s)
                     반전 강도 '강'이면 카드 scale 1.0→1.04→1.0 (ease-snap) + 라이트 플레어
[서사]   1.70–2.90s  실명(display-l) 하단서 상승 y:16→0 opacity 0→1 (0.4s, delay 0.1)
                     에피셋 타이핑/페이드(0.5s), teamResult 배지 스탬프-인(scale 1.3→1)
                     하단 진행 스트립에 현재 슬롯 칩 점등
[퇴장]   2.90–3.20s  카드 x:0→-60 opacity→0 (0.3s), 스포트라이트 다음 위치로 pan
```

**핵심 모션 원칙**:
- 90° 지점 컨텐츠 스왑으로 "뒷면이 안 보이다 나타나는" 마술감. `transform-style: preserve-3d`, `backface-visibility: hidden` 두 면.
- 익명면 brightness 감소로 "빛이 꺼지고" → 공개면 앰버로 "다시 켜짐" = 감정 전환의 시각 은유.
- 반전 '강' 카드만 라이트 플레어/스케일 펀치 추가 → 클라이맥스 차등(전부 강하면 강한 게 없다).
- 사운드 없음 전제: 홀드(정적 0.5s)가 사운드 대신 긴장을 만든다. 침묵의 페이싱이 곧 드라마.

### 4.4 스포트라이트 조명 (SVG/CSS)

```css
/* 무대 스포트라이트: 카드 뒤 radial glow */
.reveal-stage::before {
  background: radial-gradient(circle 340px at 50% 42%,
    rgba(232,176,75,0.14) 0%, rgba(232,176,75,0.05) 40%, transparent 70%);
}
```
콘 각도/위치는 카드 이동에 따라 Framer로 `x` 트윈. 나머지 화면은 `bg-vignette`로 암전 유지.

## 8. 구현 핸드오프 노트

- **토큰**: 위 §1을 `src/index.css`의 `:root` CSS 변수로, Tailwind는 `theme.extend`에서 이 변수 참조(`colors: { accent: 'var(--accent)' }`). 하드코딩 색 금지.
- **폰트**: `index.html`에 Google Fonts preconnect+link (Oswald 400/600/700, Gothic A1 700/900, Noto Sans KR 400/500/600/700, Roboto Mono 500/600/700). display swap.
- **Framer Motion**: 공개 연출은 `AnimatePresence` + `motion.div` variants. §4.3 타이밍을 variants delay/duration으로. **transform/opacity/filter만** 애니메이트(width/height/top 금지 = 잰크).
- **플립**: 3D 플립은 `preserve-3d` 컨테이너 + 두 면 `backface-hidden`. rotateY 트윈. 90° 스왑은 별도 상태 or backface로 자동.
- **dnd**: @dnd-kit. 슬롯 `useDroppable`(group 검증은 onDragOver에서), 카드 `useDraggable`. DragOverlay로 픽업 고스트. 규칙 위반은 onDragEnd에서 원복+토스트.
- **스파이더/히트맵**: 순수 SVG 컴포넌트(props: axes/cells). 카드 60장 동시 렌더 → 미니 히트맵은 `<rect>` 160개지만 정적, memo 처리.
- **접근성**: 포지션 4색은 색 외에 라벨(overline) 병기(색맹 대응). CTA 포커스 링 앰버.
- **성능**: 카드 그리드 가상화 불요(풀 ~60장). 공개 연출 시 비활성 카드는 `will-change: transform` 최소화.
- **목업 참조**: `docs/design/mockup-card-explore.html`(S1 카드/탐색 + 카드 앞면 실물), `docs/design/mockup-reveal.html`(S4 공개 연출 프레임 + 플립). 둘 다 실제 토큰 적용, 단일 파일. 이 픽셀·색을 기준으로 구현.
