**참고**: robots.txt가 차단을 "권고"하는 경로를 툴이 우회 접근한 형태이므로, 실제 참가 신청/제출은 반드시 대회 페이지 UI(로그인 포함 정상 경로)로 진행해야 한다. 이 API 접근은 순수 조사 목적.

## 1. 대회 주제/과제 정의

**대회명**: 월간 해커톤 — "내가 축구 감독이라면": 월드컵 전술 웹서비스 챌린지

**주제(원문)**: "실제 월드컵 데이터를 활용한 동적 인터랙션 웹서비스 구현"

**설명(원문, description 필드 전문)**:

> [대회소개]
> 축구 경기를 보면서 누구나 한 번쯤 이런 생각을 합니다. "내가 감독이라면 저 선수 안 쓰는데." "4-3-3이 아니라 3-4-3으로 가야지."
> 이번 대회에서는 참가자가 바이브 코딩 도구를 활용해 축구 감독이 되어 전술을 직접 구성하고 시각적으로 표현할 수 있는 동적 웹서비스를 설계·구현·배포합니다.
> 서비스의 구체적인 형태와 콘셉트는 팀이 자유롭게 결정하되, 직접 드래그, 클릭, 배치, 조작하며 감독의 경험을 느낄 수 있어야 합니다.
>
> [주제]
> 실제 월드컵 데이터를 활용한 동적 인터랙션 웹서비스 구현
>
> [설명]
> 본 대회는 실제 월드컵 데이터를 활용하여, 사용자가 축구 감독이 된 것처럼 전술을 직접 구성하고 조작할 수 있는 동적 인터랙션 웹서비스를 구현하는 것을 목표로 합니다.
> 서비스의 형태와 콘셉트는 자유롭게 기획할 수 있으며, 데이터 활용 방식, UX 구성, 인터랙션 요소, AI 기능 적용 여부 등을 고려하여 독창적인 서비스를 구현합니다.

## 2. 필수 요구사항 / 기능 스펙

**중요한 구분**: 대회 설명 안의 4가지 항목은 "예시)"로 명시되어 있으며 필수 기능 체크리스트가 아니다. 실제 명문화된 필수 요건은 아래 두 가지뿐이다.

**실제 필수 요건(원문에 명시된 것만)**:
1. "실제 월드컵 데이터"를 활용할 것
2. "직접 드래그, 클릭, 배치, 조작하며 감독의 경험을 느낄 수 있어야" 함 (정적 페이지·읽기 전용 대시보드 불가, 능동적 인터랙션 필수)

**예시(원문 그대로, 강제 아님 — 아이디어 참고용)**:
- 드래그앤드롭으로 선수를 배치하고 포메이션을 구성하는 전술 보드
- 상대팀 데이터를 분석하고 AI가 맞춤 전술을 추천해주는 서비스
- 실제 경기 데이터를 기반으로 "내가 감독이었다면" 시나리오를 시뮬레이션하는 서비스
- 경기 스탯을 보며 교체·전술 변경을 지시하는 인터랙티브 서비스

**서비스 접근성 요건**:
- 웹 브라우저에서 별도 설치 없이 직접 플레이 가능한 형태여야 함
- 별도 회원가입 또는 유료 결제 없이 심사 가능해야 함
- 외부 API 사용 시 "심사자가 별도 키 없이 확인 가능해야" 함

출처: https://daker.ai/api/hackathons/world-cup-manager-tactics-web-challenge (동일 API, 이하 별도 표기 없는 항목은 모두 이 출처)
